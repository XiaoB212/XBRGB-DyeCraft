# DyeCraft（全彩工艺）

> 一个同时支持 Fabric 和 NeoForge 的 Minecraft 模组，为游戏带来 RGB 全彩染色系统、染料植物、染色台自动化等丰富内容。

[![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](LICENSE)

## ✨ 功能特性

- **🎨 全彩染色**：支持石块、玻璃、混凝土、陶瓦、羊毛共 5 种可染色方块，任意 RGB 颜色随意染，红石触发染色与手动染色产物 NBT 完全一致。
- **🖥️ 染色台 GUI**：提供直观的交互界面，支持 HEX / RGB / HSB 输入、颜色预览、预设颜色、批量染色（每 16 块消耗 1 组染料）。
- **🔴 红石自动化**：染色台接受红石信号，会按照预设颜色自动染色，消耗对应染料并将产物送入输出槽。
- **↩️ 撤销染色**：支持最多 5 步操作撤销，物品槽变动时自动阻止，界面显示成功或失败提示。
- **📥 漏斗自动化**：顶部漏斗可向输入槽输送可染色方块，底部漏斗从输出槽提取成品，侧面漏斗分别输入红、绿、蓝染料。
- **🌱 染料作物**：红、蓝、绿三种染料花，可种植在耕地上，用骨粉催熟，成熟后只能左键破坏收获，掉落种子与植物。
- **🌍 多语言支持**：完整的中文、英文翻译。
- **📟 Jade 集成**：准星指向染色方块时，显示其颜色信息（HEX、RGB、HSB）。
- **📦 完美堆叠**：无论合成、挖掘还是创造模式，所有可染色方块都能正确堆叠。

## 📋 环境要求

| 依赖 | Fabric 版 | NeoForge 版 |
|------|-----------|-------------|
| Minecraft | 1.21.1 | 1.21.1 |
| 模组加载器 | Fabric Loader ≥ 0.19.2 | NeoForge ≥ 21.1.x |
| 核心 API | Fabric API ≥ 0.116.11 | NeoForge 自带 |
| Java | **21 推荐**（25+ 需额外步骤） | **21 推荐** |
| Jade | 可选，用于查看方块颜色 | 可选 |

> 本模组同时发布 Fabric 和 NeoForge 版本，请根据你的整合包平台选择对应 JAR。

## 🚀 快速开始

### 构建 Fabric 版本

```bash
# 1. 清理并构建
./gradlew clean build

# 2. 若系统默认 Java 版本为 25 或更高，先停止旧守护进程
./gradlew --stop

# 3. 启动测试客户端
./gradlew runClient
```

构建完成后，JAR 文件生成于 `build/libs/dyecraft-1.0.0.jar`，将其与 Fabric API 一同放入客户端 `mods` 文件夹。

### 构建 NeoForge 版本

（请确保已配置 NeoForge 开发环境）

```bash
# 使用 NeoForge 专用 Gradle 任务
./gradlew build
# 或使用 runClient 测试
./gradlew runClient
```

NeoForge 版无需额外安装 API，直接将生成的 JAR 放入模组目录即可。

## 📖 详细使用指南

### 🔧 合成配方

#### RGB 混色染料

| 合成方式 | 材料 | 产物 |
|----------|------|------|
| 无序合成 | 红染料 + 绿染料 + 蓝染料 | 1 个 RGB 混色染料 |

#### 染料种子

| 种子类型 | 合成材料 | 摆放方式 |
|----------|----------|----------|
| 红色染料种子 | 虞美人 + 红染料 + 小麦种子 | 3×3 十字形 |
| 蓝色染料种子 | 矢车菊 + 蓝染料 + 小麦种子 | 3×3 十字形 |
| 绿色染料种子 | 铃兰 + 绿染料 + 小麦种子 | 3×3 十字形 |

#### 可染色方块

**核心配方（以可染色石块为例）：**

```
SSS
SDS
SSS
```

- `S` = 平滑石英块
- `D` = RGB 混色染料
- **产物**：8 个可染色石块（白色，可后续染色）

**其他变体**：将 `S` 替换为对应材料即可：

| 方块类型 | 核心材料 |
|----------|----------|
| 可染色玻璃 | 白色染色玻璃 |
| 可染色混凝土 | 白色混凝土 |
| 可染色陶瓦 | 白色陶瓦 |
| 可染色羊毛 | 白色羊毛 |

> 合成产物自带颜色数据，初始为白色（#FFFFFF），可与任何白色可染色方块堆叠。

#### 染色台

```
DCD
OWO
OOO
```

- `C` = 钻石
- `D` = RGB 混色染料
- `O` = 哭泣的黑曜石
- `W` = 工作台

#### 植物分解

| 材料 | 产物 |
|------|------|
| 红色染料植物 | 2 个红色染料 |
| 蓝色染料植物 | 2 个蓝色染料 |
| 绿色染料植物 | 2 个绿色染料 |

### 🌻 染料花种植

1. 将染料种子种在耕地上。
2. 未成熟时可使用骨粉催熟（生长阶段 0→1→2→3，约 2~3 次成熟）。
3. 成熟后**无法右键收获**，只能破坏方块才能获得掉落物：
   - **种子**：掉落 1~3 个对应种子（概率：1 个 40%、2 个 40%、3 个 20%）
   - **植物**：固定掉落 1 个对应染料植物
4. 未成熟破坏只掉落 1 个种子（无植物）。
5. 堆肥概率：**种子 40%**，**染料植物 70%**。

### 🖥️ 染色台使用教程

**界面布局**

- **上方 3 格**：输入槽，放入可染色方块。
- **下方 3 格**：输出槽，染色产物自动进入，不允许手动放入。
- **底部红/绿/蓝染料槽**：放入原版对应染料。
- **右侧控制面板**：包含 HEX/RGB/HSB 输入框、颜色预览、三个功能按钮。

**操作步骤**

1. **放置方块**：将可染色方块放入输入槽。
2. **补充染料**：在染料槽放入红、绿、蓝染料，每 16 个方块消耗每种染料 1 个。
3. **调节颜色**：
   - 直接输入 HEX 值（自动补 `#`）
   - 调整 RGB 或 HSB 滑块
   - 点击「**预设**」保存当前颜色为默认，下次打开或红石触发染色将使用该颜色
4. **手动染色**：点击「**染色**」按钮立即处理输入槽中的方块。
5. **红石染色**：对染色台施加红石信号，会自动按预设颜色染色，适合自动化流水线。
6. **撤销**：误操作时可点击「**撤销**」（最多 5 步），界面会提示成功或失败原因（例如物品槽被改动则无法撤销）。

### 📟 漏斗自动化

- **顶部漏斗** → 将可染色方块送入输入槽。
- **底部漏斗** → 从输出槽提取已染色成品。
- **侧面漏斗**（非上下方向） → 向对应的红、绿、蓝染料槽输入对应染料。

结合红石信号，可以实现全自动批量染色！

### 📟 Jade 信息显示

安装 Jade 后，准星指向任何可染色方块，屏幕上方会显示：

```
可染色玻璃 #FF0000
RGB: 255, 0, 0
HEX: #FF0000
HSB: 0°, 100%, 100%
```

## ❓ 常见问题

### 启动客户端时提示 `Unrecognized option: --add-opens`

Gradle 守护进程可能使用了 Java 25+，而 `--add-opens` 在 JDK 25 中已被移除。  
**解决方案**：

1. 停止所有守护进程：
   ```bash
   ./gradlew --stop
   ```
2. 确保 `JAVA_HOME` 指向 JDK 21：
   ```bash
   # Windows
   set JAVA_HOME=D:\Java\jdk-21
   # macOS / Linux
   export JAVA_HOME=/path/to/jdk-21
   ```
3. 重新运行。

### 如何安装到普通客户端？

根据你的平台下载对应版本：
- **Fabric**：将 `dyecraft-fabric-1.0.0.jar` 与 Fabric API 放入 `mods` 文件夹。
- **NeoForge**：将 `dyecraft-neoforge-1.0.0.jar` 直接放入 `mods` 文件夹。

确保安装了对应版本的模组加载器。

### 开发环境如何使用？

1. 克隆仓库，在 IntelliJ IDEA 中导入项目。
2. 运行 `./gradlew genSources`（Fabric）或等效任务生成源代码。
3. 启动 `runClient` 测试。
